漏洞介绍：html注入是指用户在输入端输入可执行的HTML代码，而服务端没有对该输入进行合理的过滤，导致浏览器解析执行HTML代码。漏洞危害：严重

  • Reflected（POST）

POST和GET在注入方式上没有本质上的不同，二者的区别在于参数传递的方式，GET方式是将输入的内容以?arg1=value1 and arg2=value2….的方式拼接到请求的url后面传递给服务器，而POST方式是将参数放在请求的在请求体中，只有通过抓包的方式才能看见。可以在form等元素的method属性中定义参数的传递方式。

  • iframe injection

    进入到页面后可以猜测该网页是利用iframe将robots.txt文件展示在页面上，robots.txt是在bwapp存储路径下的资源文件，那么可想而知我们如果将robots.txt替换为其他文件就可以做到任意文件的读取！

如何搭建免费的个人博客

使用Github+NodeJS+Hexo

1.环境搭建

  • 下载Git Bash

    • 使用Git完成对文章的推送，支持不同的操作系统，下载地址 。下面以windows系统为例。

  • 安装步骤：双击下载好的文件，按照提示安装即可

    • 安装完成后打开命令行或单击右键选择 Git Bash Here,输入 git --version查看是否安装成功。

  • 下载NodeJS

    • Hexo是基于NodeJS环境的静态博客，所以需要先下载NodeJS 下载地址

  • 安装步骤：双击下载好的文件，按照提示完成安装，注意：在Custom Setup 这一步选择Add to PATH,这样可以不用再去配置环境变量

  • 安装完成后在命令行或GitBash中输入命令node -v查看是否安装成功。

  • 下载Hexo

    • 接下来便是安装主角Hexo了

    • 选择你想要存放你未来博客的文件夹，然后输入命令 npm install hexo-cli -g

    • 安装完成后，输入hexo -v查看是否安装成功

    • 接下来使用hexo init进行初始化操作，完成之后打开安装文件夹出现以下目录：

      • .deploy_git存放的是发布的项目

      • node_modules是相关的依赖包

      • public是存放生成的页面

      • scaffolds存放的是生成文章等的模板

      • source存放的是用命令创建的各种文章

      • themes存放博客的主题，博客的主题是可以自己定义的

      • _config.yml:整个博客的配置文件

      • db.json:解析source中的文件得到的

      • package.json项目所需模块的配置信息

2.关联到GitHub

  • 要让我们的博客可以被外界访问到，就需要借助GitHub。

    • 首先需要注册GitHub账号

    • 创建账号成功之后在首页点击 Repositories旁边的New按钮，创建一个新的仓库

    • 创建仓库需要输入仓库的名字，格式是username.github.io，这里的username必须与前面的Owner（也就是与你的用户名一致），然后点击创建即可，之后就可以通过这个域名来访问你的博客。

    • 然后在GitBash中配置你的账户信息，将下图中双引号的内容替换为自己的用户名的邮箱

    • 为增加安全性，我们也创建SSH密钥。在GitBash中shurussh-keygen -t rsa -C "填入你的邮箱",生成过程中会让你输入存放密钥的路径，可按回车选择默认路径。随后在路径中找到id_rsa.pub文件，打开复制文件中的内容，在github的settings中选择SSH and GPG keys，点击New SSH key，title可以随便取，将复制的内容粘贴上去即可。

    • 完成之后在GitBash中输入ssh -T git@github.com,输入密码进行验证

3.搭建个人博客

  • 修改配置文件

    • 打开hexo的安装目录，修改_config.yml的内容。注意：冒号之后要加一个英文空格

    • 修改之后保存，然后在bash中分别执行以下命令。然后打开浏览器输入http://localhost:4000即可访问hexo服务器

      • hexo clean
      • hexo generate
      • hexo server

  • 将博客部署到github

    • 在bash中分别执行以下命令

      • hexo clean
      • hexo generate
      • hexo deploy(在部署的过程中需要输入gtihub的账号和密码)

    • 部署完成后在浏览器中输入https://之前设置的域名就可以访问你的个人博客了！
