BWAPP学习笔记（三）

SQL注入

  • 漏洞介绍：

    SQL注入是指把恶意够早啊的SQL命令插入到web表单提交、域名、页面请求字符串中，最终达到欺骗服务器，非法获取数据库数据的目的

  • 原理：

    例如，登录时服务器要校验用户的登录名与密码，SQL语句为SELECT * RROM user WHERE username='username' AND password='pwd'其中username和pwd为用户输入的账号密码，此时如果对用户的输入没用进行合理的过滤，那么只需要构造所谓的“万能密码”'or 1#,SQL语句就会变成SELECT * RROM user WHERE username=''or 1#'AND password='pwd',’’ or 1的结果为true，而#是SQL中的注释符，这样也就成功绕过密码进行登录。

  • 分类：

    SQL注入的分类有很多，按照数据类型可以分为数字型、字符型和搜索型；按照提交方式可以分为GET型、POST型、Cookie型的HTTP请求头注入；按照执行结果可以分为报错注入、联合查询注入、盲注和联合查询注入，盲注又可以分为基于时间的盲注和基于布尔的盲注

  • BWAPP中的SQL注入：

    1.GET/Search

可以看到页面实现的是输入电影名进行查询，而且参数是以GET的方式传递。在输入后面加一个单引号来简单判断注入点。

从执行的结果来看这应该是基于报错的注入，并且从报错信息的“%”我们可以猜测执行的SQL语句是SELECT * FROM movies WHERE title like '%title%',所以我们的任务就是将’%进行闭合，输入IRON MAN %’ or 1=1#,这样SQL语句就变为SELECT * FROM movies WHERE title like '%IRON MAN %' or 1=1#%'，就等于是查询了movies表中的所有数据，查看执行结果：

接下来判断可显示的字段：123%' union select 1,2,3,4,5,6,7#

说明我们可以在2,3,5,4处插入我们的SQL语句。接下来查看当前的数据库和用户123%' union select 1,user(),3,database(),5,6,7#

然后我们选择查看login和password字段下的值123%' union select 1,(select group_concat(login,'-',password,'-') from users limit 0,1),3,4,5,6,7 #

这样我们就查询到了表中前两名用户的用户名和密码，显然密码是MD5加密过的，使用MD5解密即可。在medium和high等级下它使用了addslashes（）和mysql_real_escape_string（）函数对输入进行了过滤，对单双引号、反斜杆等符号进行了转义，有兴趣的读者可以自行尝试。

如何搭建个人博客

2020-03-21

如何搭建免费的个人博客

使用Github+NodeJS+Hexo

1.环境搭建

  • 下载Git Bash

    • 使用Git完成对文章的推送，支持不同的操作系统，下载地址 。下面以windows系统为例。

  • 安装步骤：双击下载好的文件，按照提示安装即可

    • 安装完成后打开命令行或单击右键选择 Git Bash Here,输入 git --version查看是否安装成功。

  • 下载NodeJS

    • Hexo是基于NodeJS环境的静态博客，所以需要先下载NodeJS 下载地址

  • 安装步骤：双击下载好的文件，按照提示完成安装，注意：在Custom Setup 这一步选择Add to PATH,这样可以不用再去配置环境变量

  • 安装完成后在命令行或GitBash中输入命令node -v查看是否安装成功。

  • 下载Hexo

    • 接下来便是安装主角Hexo了

    • 选择你想要存放你未来博客的文件夹，然后输入命令 npm install hexo-cli -g

    • 安装完成后，输入hexo -v查看是否安装成功

    • 接下来使用hexo init进行初始化操作，完成之后打开安装文件夹出现以下目录：

      • .deploy_git存放的是发布的项目

      • node_modules是相关的依赖包

      • public是存放生成的页面

      • scaffolds存放的是生成文章等的模板

      • source存放的是用命令创建的各种文章

      • themes存放博客的主题，博客的主题是可以自己定义的

      • _config.yml:整个博客的配置文件

      • db.json:解析source中的文件得到的

      • package.json项目所需模块的配置信息

2.关联到GitHub

  • 要让我们的博客可以被外界访问到，就需要借助GitHub。

    • 首先需要注册GitHub账号

    • 创建账号成功之后在首页点击 Repositories旁边的New按钮，创建一个新的仓库

    • 创建仓库需要输入仓库的名字，格式是username.github.io，这里的username必须与前面的Owner（也就是与你的用户名一致），然后点击创建即可，之后就可以通过这个域名来访问你的博客。

    • 然后在GitBash中配置你的账户信息，将下图中双引号的内容替换为自己的用户名的邮箱

    • 为增加安全性，我们也创建SSH密钥。在GitBash中shurussh-keygen -t rsa -C "填入你的邮箱",生成过程中会让你输入存放密钥的路径，可按回车选择默认路径。随后在路径中找到id_rsa.pub文件，打开复制文件中的内容，在github的settings中选择SSH and GPG keys，点击New SSH key，title可以随便取，将复制的内容粘贴上去即可。

    • 完成之后在GitBash中输入ssh -T git@github.com,输入密码进行验证

3.搭建个人博客

  • 修改配置文件

    • 打开hexo的安装目录，修改_config.yml的内容。注意：冒号之后要加一个英文空格

    • 修改之后保存，然后在bash中分别执行以下命令。然后打开浏览器输入http://localhost:4000即可访问hexo服务器

      • hexo clean
      • hexo generate
      • hexo server

  • 将博客部署到github

    • 在bash中分别执行以下命令

      • hexo clean
      • hexo generate
      • hexo deploy(在部署的过程中需要输入gtihub的账号和密码)

    • 部署完成后在浏览器中输入https://之前设置的域名就可以访问你的个人博客了！
